🌐 PORT/LOCALHOST ELIMINATION - Production Deployment Ready

## ✅ CRITICAL FIX - Deployment Blocker Resolved

Eliminated all hardcoded localhost and port assumptions from production code.
biomeOS can now deploy to any environment without code changes.

### Production Violations Eliminated (18 → 0)

1. ✅ WebSocket/SSE endpoints - environment-based discovery
2. ✅ Discovery endpoint fallbacks - no localhost assumptions  
3. ✅ Network binding - configurable via environment
4. ✅ Port numbers - dynamic allocation support
5. ✅ Test configuration - environment-driven

### Philosophy: Socket-First, Network-Optional

Unix sockets are now the preferred transport:
- Faster (no TCP overhead)
- More secure (filesystem permissions)
- Local-only by default
- No port conflicts

HTTP/network only when needed for remote access.

### Environment Variables (New)

Discovery:
- BIOMEOS_DISCOVERY_ENDPOINT (unix:// or http://)
- BIOMEOS_WS_ENDPOINT (WebSocket events)
- BIOMEOS_SSE_ENDPOINT (Server-Sent Events)

Network:
- BIOMEOS_BIND_ADDRESS (default: 127.0.0.1 in dev)
- BIOMEOS_PORT (default: 8080 in dev)
- BIOMEOS_TEST_BIND, BIOMEOS_TEST_PORT (for tests)

### Impact

Before:
❌ Hardcoded ws://localhost:8080 - fails in production
❌ Localhost fallback in discovery - assumes local deployment
❌ Fixed ports - conflicts in multi-instance setups

After:
✅ Environment-based endpoints - works anywhere
✅ Explicit failure if misconfigured - no silent assumptions
✅ Dynamic port allocation - no conflicts

### Migration

Development:
```bash
export BIOMEOS_DISCOVERY_ENDPOINT=unix:///tmp/songbird.sock
export BIOMEOS_WS_ENDPOINT=ws://localhost:8080/ws
```

Production:
```bash
export BIOMEOS_DISCOVERY_ENDPOINT=unix:///var/run/biomeos/discovery.sock
export BIOMEOS_BIND_ADDRESS=0.0.0.0
export BIOMEOS_PORT=443
export BIOMEOS_WS_ENDPOINT=wss://events.biomeos.io/ws
```
