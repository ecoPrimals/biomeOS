## 🆕 **New Commands Added**

### **🚀 Deployment & Lifecycle Management**
```bash
# Deploy biomes from YAML manifests
biomeos deploy my-biome.yaml --environment production --watch

# Create new biomes from templates or niches
biomeos create my-app --niche gaming --output ./biomes
biomeos create my-ml-biome --niche ai --output ./projects
biomeos create custom-biome --blank --output ./custom

# Dry-run validation
biomeos deploy my-biome.yaml --dry-run
```

### **📜 Log Management & Debugging**
```bash
# Stream logs from services
biomeos logs web-service --follow --lines 100
biomeos logs api-gateway --since "2h" --lines 50

# Real-time log following
biomeos logs database --follow
```

### **💻 Service Execution**
```bash
# Execute commands in running services
biomeos exec web-service -- ps aux
biomeos exec database --interactive -- bash
biomeos exec api-service --workdir /app -- npm test
```

### **⚖️ Dynamic Scaling**
```bash
# Scale services up/down
biomeos scale web-service --replicas 10
biomeos scale api-gateway --cpu "4 cores" --memory "8GB"
biomeos scale database --replicas 3 --cpu "2 cores"
```

### **🤖 AI-Powered Assistance**
```bash
# Natural language assistance
biomeos ai "How do I deploy a gaming biome?"
biomeos ai "Scale my web service for high traffic" --explain
biomeos ai "Debug connection issues" --execute

# AI reasoning and explanations
biomeos ai "Set up monitoring for my cluster" --explain
```

## 🚀 **Real-World Usage Examples**

### **Complete Biome Deployment Workflow**
```bash
# 1. Create a new gaming biome
biomeos create gaming-server --niche gaming --output ./biomes

# 2. Deploy with validation
biomeos deploy ./biomes/gaming-server.biome.yaml --dry-run
biomeos deploy ./biomes/gaming-server.biome.yaml --watch

# 3. Monitor and scale
biomeos health --system --watch 30
biomeos scale game-engine --replicas 5 --cpu "4 cores"

# 4. Debug and troubleshoot
biomeos logs game-engine --follow
biomeos exec game-engine --interactive -- bash

# 5. AI assistance
biomeos ai "Optimize my gaming server for 1000 concurrent players" --explain
```

### **Development & Debugging Workflow**
```bash
# Quick status check
biomeos status --detailed --services --resources

# Stream logs for debugging
biomeos logs api-service --follow --since "1h"

# Execute diagnostic commands
biomeos exec database -- pg_stat_activity
biomeos exec web-server -- curl localhost:8080/health

# AI-assisted troubleshooting
biomeos ai "My API is responding slowly, help me debug" --explain
```
