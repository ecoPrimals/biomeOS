## 💡 Key Concepts

### Zero-Coupling Architecture
biomeOS discovers everything at runtime:
- **Services** - What exists, where it is
- **Capabilities** - What each service can do
- **APIs** - How to communicate with services
- **Operations** - How to invoke functionality

### Dynamic API Discovery
Instead of hardcoded clients, biomeOS:
1. Fetches OpenAPI spec from `/api/schema`
2. Creates appropriate adapter
3. Calls operations dynamically
4. Adapts to API changes automatically

### Sovereignty-Respecting
- Users choose implementations
- Primals control their APIs
- No forced coupling
- Auditable and transparent
