## рџЋЇ CORE PRINCIPLES (From Specs)

### 1. biomeOS is a Primal

- Not special, not privileged
- Specializes in ecosystem management
- Can be nested, replaced, or shut down
- Follows same lifecycle rules as all primals

### 2. No Hardcoding (TRUE PRIMAL)

- Every primal learns its environment
- Infant discovery at runtime
- Capability-based discovery
- Event-driven coordination

### 3. Security Inheritance

- All primals inherit from Tower Atomic
- Lineage traced back to BearDog
- Nested primals inherit from parents
- Genetic security model

### 4. Lifecycle Management

- **Germination**: Birth with minimal knowledge
- **Terraria**: Safe learning environment
- **Imprinting**: Ecosystem structure transfer
- **Injection**: Live ecosystem introduction
- **Infant Discovery**: Continuous runtime learning
- **Apoptosis**: Graceful shutdown (future)

### 5. Nested Environments

- biomeOS can be nested
- Sub-niches for testing, development, staging
- Each nested biomeOS is also a primal
- Communication via RPC (no special access)
