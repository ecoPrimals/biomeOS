## 🧬 Core Concept: Self-Hosting Evolution

**Traditional Systems:**
- Developer writes code → compiles → deploys
- User uses pre-defined features
- Evolution happens outside the system

**biomeOS with Squirrel + petalTongue:**
- User sees NUCLEUS in 3D (petalTongue)
- User asks Squirrel to optimize/expand
- Squirrel generates Neural API graph
- User approves in petalTongue UI
- Neural API deploys new niche
- **System evolves itself!** ✨

---

## 🎨 petalTongue Role

### **Visualization & Interaction**

**What petalTongue Shows:**
1. **Live NUCLEUS Topology** (3D graph)
   - Primals as nodes (BearDog, Songbird, Toadstool, NestGate)
   - Connections as edges (Unix sockets, data flow)
   - Health status (colors, animations)
   - Resource usage (size, glow)

2. **Available Niches** (Template library)
   - Tower, Node, Nest atomics
   - Custom workloads
   - Neural API graphs

3. **Deployment Interface** (User bootstrapping)
   - Drag-and-drop graph nodes
   - Configure parameters (family ID, resources)
   - Preview deployment plan
   - Execute via Neural API

**Key Features:**
- **3D rendering** via Toadstool (GPU acceleration)
- **Real-time updates** (WebSocket to biomeOS)
- **Multi-modal** (GUI for visual, TUI for terminal, headless for automation)

---

## 🐿️ Squirrel Role

### **AI Coordination & Agentic Systems**

**What Squirrel Does:**

1. **System Analysis**
   - Query NUCLEUS health (via biomeOS clients)
   - Analyze resource usage (Toadstool metrics)
   - Identify optimization opportunities

2. **Graph Generation**
   - User: "I need a development environment"
   - Squirrel: Generates Neural API graph
     ```toml
     # dev_environment.toml
     [[nodes]]
     id = "jupyter"
     node_type = "primal.start"
     dependencies = ["toadstool"]
     # ... generated by Squirrel
     ```

3. **Agentic Workflows**
   - Monitor NUCLEUS continuously
   - Suggest optimizations
   - Auto-scale based on load
   - Predictive resource allocation

4. **Natural Language → Graphs**
   - User describes what they want
   - Squirrel translates to Neural API TOML
   - petalTongue visualizes the plan
   - User approves → Neural API executes

## 🔄 Integration Workflow

### **User Bootstraps a New Niche**

**Step 1: User Interaction (petalTongue)**
```
User in petalTongue 3D view:
  1. Sees current NUCLEUS (Tower, Node, Nest)
  2. Opens "Deploy New Niche" menu
  3. Types: "I want to run Jupyter with GPU support"
```

**Step 2: AI Analysis (Squirrel)**
```rust
// Squirrel receives request
let request = UserRequest {
    description: "Jupyter with GPU support",
    context: nucleus_topology,
};

// Squirrel analyzes
let analysis = squirrel.analyze_request(&request).await?;
// Result: Needs Toadstool (GPU), NestGate (persistence), Network access

// Squirrel generates graph
let graph = squirrel.generate_graph(&analysis).await?;
```

**Step 3: Visualization (petalTongue)**
```
petalTongue shows:
  - Current NUCLEUS (existing nodes)
  - Proposed Jupyter niche (new node)
  - Dependencies (edges to Toadstool, NestGate)
  - Resource requirements (GPU, 4GB RAM)
  
User sees:
  ┌─────────────┐
  │  BearDog    │─┐
  └─────────────┘ │
  ┌─────────────┐ │    ┌──────────────┐
  │  Songbird   │─┼───►│  Toadstool   │──┐
  └─────────────┘ │    └──────────────┘  │
  ┌─────────────┐ │    ┌──────────────┐  │   ┌─────────────┐
  │  NestGate   │─┼───►│   Jupyter    │◄─┘   │  NEW!       │
  └─────────────┘ │    └──────────────┘      │  GPU: ✅    │
                  │                           │  RAM: 4GB   │
                  │                           └─────────────┘
```

**Step 4: User Approval**
```
petalTongue UI:
  ┌──────────────────────────────────────────┐
  │ Deploy Jupyter Niche?                    │
  │                                          │
  │ Resources:                               │
  │   • GPU: 1x (via Toadstool)              │
  │   • RAM: 4GB                             │
  │   • Storage: 10GB (via NestGate)         │
  │                                          │
  │ Dependencies:                            │
  │   ✅ Toadstool (compute)                 │
  │   ✅ NestGate (persistence)              │
  │   ✅ BearDog (security)                  │
  │                                          │
  │ Estimated time: 15 seconds               │
  │                                          │
  │  [ Deploy ]  [ Customize ]  [ Cancel ]   │
  └──────────────────────────────────────────┘
```

**Step 5: Neural API Execution**
```bash
# User clicks "Deploy"
# petalTongue sends to biomeOS:
{
  "method": "neural_api.execute_graph",
  "params": {
    "graph": squirrel_generated_jupyter_graph,
    "family_id": "nat0"
  }
}

# Neural API deploys
nucleus execute --graph /tmp/jupyter_niche.toml --family nat0
```

**Step 6: Live Feedback (petalTongue)**
```
petalTongue real-time updates:
  Phase 1/3: Verifying resources... ✅ (2s)
  Phase 2/3: Deploying Jupyter...   ✅ (8s)
  Phase 3/3: Health check...        ✅ (3s)
  
  Total: 13 seconds
  
  Jupyter running at: /tmp/jupyter-nat0.sock
  3D view updates to show new node! 🎉
```

## 🧠 Neural API Integration

### **How They Use Neural API**

**1. petalTongue: Graph Visualization**
```rust
// petalTongue queries available graphs
let graphs = biomeos_client.list_graphs().await?;

// User selects a graph
// petalTongue visualizes it in 3D
for node in graph.nodes {
    render_3d_node(node.id, node.dependencies, node.config);
}

// User can edit graph visually
// Drag nodes, add dependencies, configure params
```

**2. Squirrel: Graph Generation**
```rust
// Squirrel creates graphs programmatically
let graph = Graph {
    id: "user-requested-niche",
    nodes: vec![
        GraphNode {
            id: "new-service",
            node_type: "primal.start",
            dependencies: vec!["toadstool"],
            config: squirrel_analyzed_config,
        }
    ],
};

// Save to Neural API
biomeos_client.save_graph(&graph).await?;
```

**3. User: Graph Execution**
```rust
// Via petalTongue UI, user triggers:
biomeos_client.execute_graph("user-requested-niche", "nat0").await?;

// petalTongue shows live progress
// Squirrel monitors and provides insights
```

```toml
# UI Atomic: Squirrel + petalTongue on NUCLEUS
# User/Agentic layer for self-hosted evolution

[graph]
id = "ui-atomic"
description = "Deploy user interface and AI layer on NUCLEUS"
version = "1.0.0"

# Prerequisite: NUCLEUS must be running
[[nodes]]

[[nodes]]
id = "petaltongue"
node_type = "primal.start"
dependencies = ["verify-nucleus"]

binary_path = "plasmidBin/primals/petaltongue"

[nodes.config]

    "/tmp/squirrel-${FAMILY_ID}.sock",
    "/tmp/petaltongue-${FAMILY_ID}.sock"
```

## 🔌 API Surface for Integration

### **biomeOS Provides to Squirrel + petalTongue**

**1. Neural API Client**
```rust
// In biomeos-core/src/clients/neural_api.rs
pub struct NeuralApiClient {
    // ...
}

impl NeuralApiClient {
    // List available graphs
    pub async fn list_graphs(&self) -> Result<Vec<GraphMetadata>>;
    
    // Get graph details
    pub async fn get_graph(&self, id: &str) -> Result<Graph>;
    
    // Save user-created graph
    pub async fn save_graph(&self, graph: &Graph) -> Result<()>;
    
    // Execute graph
    pub async fn execute_graph(&self, id: &str, family_id: &str) -> Result<ExecutionHandle>;
    
    // Monitor execution
    pub async fn get_execution_status(&self, handle: &ExecutionHandle) -> Result<ExecutionStatus>;
    
    // Cancel execution
    pub async fn cancel_execution(&self, handle: &ExecutionHandle) -> Result<()>;
}
```

**2. Topology API**
```rust
// Real-time NUCLEUS topology
pub async fn get_topology(&self) -> Result<Topology> {
    Topology {
        primals: vec![
            PrimalNode {
                id: "beardog-nat0",
                type: "security",
                socket: "/tmp/beardog-nat0-default.sock",
                health: HealthStatus::Healthy,
                connections: vec!["songbird-nat0"],
            },
            // ... all primals
        ],
        connections: vec![
            Connection {
                from: "songbird-nat0",
                to: "beardog-nat0",
                type: "security-provider",
            },
            // ... all connections
        ],
    }
}
```

**3. Deployment API**
```rust
// Deploy niche from UI
pub async fn deploy_niche(&self, niche: &NicheTemplate) -> Result<DeploymentHandle>;

// Get available templates
pub async fn list_niche_templates(&self) -> Result<Vec<NicheTemplate>>;
```
