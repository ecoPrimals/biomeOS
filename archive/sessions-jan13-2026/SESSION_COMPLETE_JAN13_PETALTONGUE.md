### 2. Port-Free is the TRUE PRIMAL Way

**Why Unix Sockets > TCP Ports**:

**Security**:
- Unix sockets = filesystem permissions (chmod 600)
- TCP ports = network exposure, firewalls, scanning

**Performance**:
- Unix sockets = kernel direct copy (~10x faster)
- TCP ports = full TCP/IP stack overhead

**Simplicity**:
- Unix sockets = no conflicts, zero config
- TCP ports = port conflicts, firewall rules, NAT

**Discovery**:
- Unix sockets = scan `/run/user/<uid>/*.sock`
- TCP ports = hardcoded or complex service discovery
