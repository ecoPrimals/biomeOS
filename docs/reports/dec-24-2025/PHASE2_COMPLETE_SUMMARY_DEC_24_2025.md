## 💡 **Key Technical Decisions**

### 1. Capability-Based Discovery
**Decision**: Services discovered by capability, not hardcoded names  
**Validation**: Verified through 50+ tests  
**Impact**: System is flexible and extensible

### 2. Zero-Knowledge Startup
**Decision**: Primals start with no hardcoded dependencies  
**Implementation**: `DiscoveryBootstrap` module  
**Impact**: True service independence

### 3. Delegation Over Reimplementation
**Decision**: BiomeOS delegates to specialized primals  
**Examples**: Songbird for discovery, ToadStool for compute  
**Impact**: Clear boundaries, maintainable code

### 4. Mock Servers for Testing
**Decision**: Use `wiremock` for HTTP testing  
**Implementation**: All integration tests use mocks  
**Impact**: Fast, reliable, reproducible tests
