## 💡 Key Insights to Remember

### 1. Ecological vs Molecular

**Critical architectural distinction**:

**Ecological** (within system):
```
Basement HPC System:
- BearDog, Songbird, ToadStool, NestGate, Squirrel
- All interact ecologically (symbiotic)
- Same environment, shared resources
- Free, instant communication
```

**Molecular** (between systems):
```
Your HPC ↔ Friend's HPC: Covalent (high trust)
Your HPC ↔ Cloud: Ionic (contract-based)
Cloud's internal GPU farm: Metallic (specialized)
```

**Primals don't know about bonding** - systems do!

### 2. UniBin Universality

Same binary works everywhere:
```bash
# Local deployment
SONGBIRD_SOCKET=/tmp/songbird-local.sock
./songbird server

# Cloud deployment (same binary!)
SONGBIRD_SOCKET=/tmp/songbird-cloud.sock
METERING_ENABLED=true
./songbird server
```

**Adapts based on environment, no upper-layer knowledge!**

### 3. Deep Debt > Quick Fixes

**Always choose strategic over tactical**:
```
Problem: DAG bug
Quick Fix: ❌ Debug for hours
Deep Debt: ✅ Pin + evolve

Result: Team productive + proper solution
```

### 4. Routing Priority

```
1. Ecological (same system) → Free, instant
2. Covalent (trusted system) → Free, fast
3. Ionic (contract system) → Metered, costs $$$
```

**This is automatic** - primals discover at runtime!
